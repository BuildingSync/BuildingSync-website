Mapping Use Cases

The diagram below shows the process of adding a use case to the selection-tool website. Tips for filling out the mapping spreadsheet can also be found below.

Tips for Mapping Use Cases

  • The mapping spreadsheet has 6 columns:
    • BuildingSyncPath — This column is auto-generated when download the mapping spreadsheet for your use case. It contains a row for each element in the BuildingSync schema.
    • State — The state column indicates whether an element of the schema is required or optional for the use case. Leave blank when an element is not applicable to the use case. Possible state values are:
      • Required — This element is required for the use case.
      • Optional — This element is optional for the use case.
      • Required-1 — This is used in conjunction with the GroupID and GroupingLevel columns. It is used to indicate that one (and only one) element in the group defined by groupID is required.
      • Optional-1 — This is used in conjunction with the GroupID and GroupingLevel columns. It is used to indicate that, while optional, if one of the elements in the group (defined by groupID) is present, then there can be only one.
    • RequiredValues — This column should contain a comma-delimited list of the allowable values for the element.
    • RequiredPairedElement — If the element has associated elements, list the paths of the other elements here, in comma-delimited format. Use the same syntax as in the BuildingSyncPath column.
    • GroupID — This column is only required to designate groups of elements where only one element is required or optional. The ID should be unique across the use case. Leave blank otherwise.
    • GroupingLevel — This column is required for groups of elements where only one element is required or optional. It is used to mark elements in the path appropriately, as required or optional. The level should be the lowest level under which there is a required or optional element. If there is a groupID, enter the grouping level using the same syntax as in the BuildingSyncPath column. Leave blank otherwise.
  • Each BuildingSyncPath can only be part of one group. The selection-tool does not support complex relationships where a path is present in multiple groups.
  • GroupID and GroupingLevel should only be used for identifying groups of elements where only one of the group is required/optional. It should not be used to designate a list of values for a specific element.
